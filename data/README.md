# Data Directory Layout

This directory holds all simulation data for the 30 FDS scenarios.

```
data/
├── raw/           Raw FDS outputs — never modify these files.
└── processed/     Normalised tensors ready for training — auto-generated.
```

---

## `data/raw/`

Contains one sub-directory per FDS scenario:

```
data/raw/
├── scenario_000/
│   ├── scenario_000.fds    FDS input deck
│   ├── scenario_000.smv    Smokeview control file
│   ├── *.sf                Slice files (temperature, visibility, CO)
│   └── *.q                 PLOT3D output (if any)
├── scenario_001/
│   └── ...
└── scenario_029/
    └── ...
```

Naming convention: `scenario_{i:03d}` where i ∈ [0, 29].

**Do not commit raw data to git** — files are large and listed in `.gitignore`.
Download from the shared storage and place here before running the pipeline.

---

## `data/processed/`

Generated by `scripts/run_data_pipeline.sh` (or `src/data_pipeline/build_dataset.py`).

```
data/processed/
└── dataset.h5           HDF5 file containing all normalised tensors.
```

HDF5 group structure:

```
/scenario_000/
    input   shape (31, 5, 60, 40, 6)   float32, normalised [0,1]
    target  shape (31, 3, 60, 40, 6)   float32, normalised [0,1]
/scenario_001/
    ...
/scenario_029/
    ...
/metadata/
    train_indices   int array of shape (24,)
    val_indices     int array of shape (3,)
    ood_indices     int array of shape (3,)
    normalization   YAML string with normalization parameters
```

Channel ordering — **input** (5 channels):

| idx | Name | Unit | Normalization |
|-----|------|------|---------------|
| 0 | Temperature | °C | `(T−20)/1180` |
| 1 | Visibility | m | `1−V/30` (inverse) |
| 2 | CO | ppm | `log1p(CO)/log1p(5000)` |
| 3 | Mask | — | 1=fluid, 0=solid |
| 4 | Time encoding | — | `t/300` |

Channel ordering — **target** (3 channels):

| idx | Name | Same normalization |
|-----|------|--------------------|
| 0 | Temperature | Yes |
| 1 | Visibility | Yes |
| 2 | CO | Yes |

## Split Assignment

Scenarios are assigned to splits once during `build_dataset.py` and stored
in `/metadata/` inside the HDF5 file. The random seed is fixed at 42.

| Split | Count | Purpose |
|-------|-------|---------|
| Train | 24 | Gradient updates |
| Val | 3 | Hyperparameter tuning |
| OOD | 3 | Out-of-distribution evaluation |
